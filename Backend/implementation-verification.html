🔄 Data Flow Overview

1. Onboarding Flow

User fills form (Job Role, Bio, Skills)
↓
Click "Complete Profile"
↓
PUT /api/profile with form data
↓
Backend validates & saves to User model
↓
Sets onboardingCompleted = true
↓
Navigate to Dashboard

2. Question Generation Flow

Frontend: generateQuestions(profileId, difficulty, count)
↓
POST /api/questions/generate
↓
Auth middleware extracts user ID from token
↓
getQuestionsForTest(profileId, difficulty, count, userId)
↓
Optional: Look up user history & filter answered Qs
↓
Return fresh question array to frontend

3. Grading & Storage Flow

Frontend: submitTest() with detailedAnswers
↓
POST /api/intelligence/evaluate
↓
evaluateProfileIntelligence() receives detailedAnswers
↓
Look up questions in QuestionBank database
↓
Compare answers (case-insensitive string match)
↓
Calculate score: (correctCount / totalCount) × 100
↓
Apply difficulty weights + profile bonuses
↓
generateUserRank() → calculateTrustScore() → assignBadge()
↓
saveEvaluationResult() to MongoDB
↓
Save TestSession & IntegrityEvent records
↓
Return complete result with sessionId & slug
